Programs using the libplayerc Python bindings will generally have the following structure: @include joydrive.py.doc The steps in this program mirror those described for standard C clients. Thus, the C documentation can be used as a guide, so long as one makes some minor mental translations; e.g.: @verbatim proxy = playerc_position_create(...); -> proxy = playerc_position(...) playerc_position_set_speed(proxy, ...); -> proxy.set_speed(...) @endverbatim Detailed information for each C proxy can be found in the Reference section. */
