@page supported_hardware Supported devices

<p>
The following tables list the hardware and software systems that are
supported by Player.  Also given are the names of the maintainers for each
driver.  However, most questions should be directed to our 
<a href="http://sourceforge.net/mail/?group_id=42445">mailing lists</a>,
so that they are archived for all to use.
</p>
<p>
These tables should generally be up to date
with respect to the current code in CVS, and thus may be newer than the
latest official release.
</p>
<p>
Please post additions and corrections to these tables to our 
<a href="http://lists.sourceforge.net/lists/listinfo/playerstage-developers">
developers' mailing list</a>.
</p>

<h4><a name="player_hardware_alg">Algorithms</a></h4>
<table border=2>
<tr>
<td>Algorithm(s)</td>
<td>Driver(s)/ Interface(s)</td>
<td>References</td>
<td>Notes</td>
<td>Maintainer</td>
</tr>

<tr>
<td>configuration space</td>
<td>lasercspace/laser</td>
<td></td>
<td>Modulates raw laser data according to the robot's dimensions in order
to represent the empty area in the robot's configuration space.</td>
<td><a href="http://robotics.usc.edu/~ahoward/">Andrew Howard</a></td>
</tr>

<tr>
<td><a href="http://www-personal.engin.umich.edu/~johannb/vff&vfh.htm">
Vector Field Histogram (VFH)</a> goal-seeking obstacle avoidance
algorithm</td>
<td>vfh/position</td>
<td>Based on <a href="http://www-personal.engin.umich.edu/~johannb/">
Johann Borenstein's</a> VFH.</td>
<td>Most parameters of the algorithm can be set in Player's configuration
file.</td>
<td><a href="http://robotics.usc.edu/~cvjones/">Chris Jones</a></td>
</tr>

<tr>
<td>
Adaptive Monte Carlo Localization (AMCL)</a> probabilistic localization
algorithm</td>
<td>amcl/localize</td>

<td>
See <a href="http://www.cs.washington.edu/homes/fox/">Dieter Fox's</a> paper on 
<a href="http://www.cs.washington.edu/homes/fox/abstracts/adaptive-sampling-nips-01.abstract.html">adaptive sampling<a>
</td>

<td>Supports localization based on many types of sensor data, including:
odometry, sonar, laser, and wifi signal strength.</td>
<td><a href="http://robotics.usc.edu/~ahoward/">Andrew Howard</a></td>
</tr>

<tr>
<td>Wavefront propagation planner</td>
<td>wavefront/position</td>

<td>
See <a href="http://robotics.stanford.edu/~latombe">Jean-Claude Latombe's</a>
authoritative book
<a href="http://search.barnesandnoble.com/booksearch/isbnInquiry.asp?userid=2WIJN3PX4U&isbn=0792391292&itm=1">Robot Motion Planning</a>
</td>

<td>
Performs configuration-space path-planning for a planar robot via cell
decomposition and wavefront propagation.  Requires an underlying
localization device (e.g., amcl) and an underlying obstacle-avoiding
position device (e.g., vfh).
</td>
<td><a href="http://www.ai.sri.com/~gerkey">Brian Gerkey</a></td>
</tr>
</table>
